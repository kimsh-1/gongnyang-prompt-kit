# v2.0 실측 검증 리포트 (2026-07-02)

54/56 이미지 생성 완료(미완 2건은 모두 s10 반도 스윔 — 아래 참조), codex `$imagegen` 병렬 6워커 + 순차 보정 러닝, 채점은 고정 루브릭(RUBRIC.md) 비전 판독. **이 리포트의 결론이 v2.0 룰북 문구를 확정한다** — 반영 완료 항목은 각 절 끝에 표기.

## 실험 A — 한글 텍스트 렌더 (24장, 2×2×2×3반복)

| 셀 | text_accuracy | 합격 |
|---|---|---|
| base·1024·medium | 4.67 | 2/3 |
| base·1024·high | 5.00 | 3/3 |
| base·2048·medium/high | 5.00 | 6/6 |
| treat·1024·medium | 5.00 | 3/3 |
| treat·1024·high | **4.00** | 2/3 |
| treat·2048·medium/high | 5.00 | 6/6 |

주효과: 프로토콜 base 4.92 vs treat 4.75 / **size 1024 4.67 vs 2048 5.00 (12/12 만점)** / quality medium 4.92 vs high 4.75.

**결론:**
1. **강화 카피 프로토콜은 개선 없음** — 기본(따옴표 고정 + 긍정 가드)이 이미 천장(11/12 만점).
2. **하이픈 스펠아웃은 유해** — treat·1024·high·r3에서 "붉-은 벽-돌"의 하이픈 7개가 글자로 렌더(주입 사고). → **룰북 후퇴 완료**: 한글 스펠아웃 금지, 영문 조어만 최후수단 (typography-layout §4, SKILL.md 텍스트 렌더).
3. **진짜 레버는 캔버스 크기** — 2048 전 반복 만점, 실패 전부 1024. quality는 유의차 없음. → **룰북 격상 완료**: "돈은 quality가 아니라 size에" (typography-layout §7).

## 실험 B — 화보 컴플라이언스 레인 (10피사체×2암, 유효 18장)

| 암 | tail 글자 렌더 | compliance | goal_fit | 명시적 refusal |
|---|---|---|---|---|
| baseline (긍정형만, n=9) | — | 4.56 | 4.89 | 0 |
| treatment (Tier-2 블록, n=9) | **0/9** | 4.44 | 4.89 | 0 |

**결론:**
1. **NEGATIVE_TAIL은 이미지에 렌더되지 않는다** (0/9) — 장면 네거티브와 달리 무해 실증. 품질 비용 없음(goal_fit 동률, compliance 차이는 의상 고유 감점). → **Tier-2 채택 확정** (editorial-hwabo §3에 실측 노트 반영 완료).
2. **거부율 효과는 N=10에서 판정 불가** — 양 암 모두 명시적 `reason=refusal` 0건. Tier-2의 채택 근거는 "무해 + 정책 의도 명시"이며, 거부율 개선 주장은 하지 않는다.
3. **s10(화이트 반도 스윔, photoreal)은 불안정 피사체** — 양 암 합산 5회 시도 전부 타임아웃(명시 거부 없음). 로컬 상세판의 보수성 사다리(swimwear photoreal 최상 엄격)와 부합.

## 실험 C — 레이아웃 정밀도 (12장, 3패턴×2암×2반복)

| 패턴 | baseline | treatment(영역 문법) |

**결론:** 영역 문법은 전 패턴에서 동률 이상, 서브 영역 정밀 준수(마진·좌측 정렬 축자 이행)에서 우세. 천장 효과로 변별력은 제한적이나 해가 없고 정밀 제어가 실증됨. → **확정 채택** (typography-layout §1 노트 반영 완료). 3×2 그리드는 양 암 모두 6셀 정확 — gpt-image-2의 그리드 카운팅은 신뢰 가능.

## 인프라 발견 — 러너 회수 레이스 (스킬 개선 대상)

병렬 워커가 `~/.codex/generated_images/` 공용 풀에서 PNG를 회수할 때 **늦게 도착한 다른 레코드의 이미지를 오인 회수**하는 레이스가 4건 발생(C 2건, B 2건 — 전부 삭제 후 순차 재생성으로 정정). 재현 조건: 타임아웃 직후 재시도와 이웃 워커 회수가 겹칠 때. **권고: codex-imagegen 러너의 회수 매칭을 세션 ID 단위로 강화하거나, 재시도는 PARALLEL=1 순차로.** (이번 실험 러너에는 순차 보정으로 대응.)
